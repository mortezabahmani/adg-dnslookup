# 🛡️ ADG-DNSLookup

**سیستم ضد-مسمومیت DNS برای AdGuard Home**

## 📖 فلسفه و بیان مسئله

در برخی مناطق، ارائه‌دهندگان خدمات اینترنت و دولت‌ها به طور فعال درخواست‌های DNS را نظارت، فیلتر یا دستکاری می‌کنند - عملی که به عنوان مسمومیت DNS (DNS Poisoning) شناخته می‌شود. وقتی شما سعی می‌کنید دامنه‌ی شناخته‌شده‌ای مانند `google.com` را resolve کنید، سرورهای DNS محلی ممکن است به جای IP جهانی واقعی، یک IP محلی و جعلی برگردانند. این کار می‌تواند منجر به مسدود شدن دسترسی، به خطر افتادن امنیت یا هدایت اجباری ترافیک شما از طریق زیرساخت‌های تحت نظارت شود.

هدف **ADG-DNSLookup** حل این مشکل با دور زدن کامل دستکاری DNS محلی است.

### 💡 راه‌حل

این سیستم از یک رویکرد توزیع‌شده و قابل‌اطمینان برای resolve امن DNS از خارج از شبکه‌ی محدودشده استفاده می‌کند و آدرس‌های IP معتبر را مستقیماً به سیستم **AdGuard Home** محلی شما تزریق می‌کند.

۱. **Resolution امن خودکار**: یک ورک‌فلوی GitHub Actions هر ۱۲ ساعت از محیطی بدون محدودیت اجرا می‌شود.
۲. **مکانیسم رأی اکثریت (Majority Vote)**: دامنه‌ها با استفاده از ارائه‌دهندگان معتبر DNS-over-HTTPS (DoH) (مانند Cloudflare، Google، Quad9) resolve می‌شوند. یک IP تنها در صورتی معتبر در نظر گرفته می‌شود که حداقل ۲ از ۳ resolver با هم توافق داشته باشند.
۳. **ذخیره‌سازی آرتیفکت**: رکوردهای IP اعتبارسنجی شده به صورت شفاف به عنوان فایل‌های JSON در این مخزن (Repository) ذخیره می‌شوند.
۴. **تزریق سمت کلاینت**: یک اسکریپت سبک کلاینت در شبکه محلی شما، این فایل‌های JSON معتبر را دریافت کرده و به صورت یکپارچه قوانین DNS Rewrite دستگاه AdGuard Home شما را آپدیت می‌کند.

از این نقطه به بعد، دستگاه‌های شبکه محلی شما به سرعت دامنه‌های مهم را به آدرس‌های IP واقعی‌شان از طریق AdGuard Home، بدون نیاز به پرس‌وجو از DNS‌های بالادستی مشکوک، resolve خواهند کرد.

## ✨ ویژگی‌ها و قابلیت‌ها

- **امن و ضد-دستکاری**: استفاده از DoH و مکانیسم اجماع (رأی اکثریت) برای حذف هرگونه نقطه شکست یا دستکاری.
- **همگام‌سازی خودکار**: بخش سمت سرور به طور خودکار هر ۱۲ ساعت آپدیت می‌شود. شما می‌توانید اسکریپت کلاینت را از طریق cron job‌ها خودکار کنید.
- **حالت‌های کاری متعدد**:
  - `Online` (پیش‌فرض): آخرین IP‌های resolve شده را به صورت امن از GitHub دریافت می‌کند.
  - `Standalone`: خود IP‌ها را از طریق DoH امن resolve می‌کند (مفید در مواقعی که خود GitHub در دسترس نیست اما VPN فعال است).
- **لیست‌های جامع پیش‌فرض**: لیست‌های دامنه‌ی دسته‌بندی‌شده و آماده استفاده (گوگل، کلودفلر، متا، مایکروسافت، شبکه‌های اجتماعی، استریمینگ، CDNها، ابزارهای توسعه‌دهندگان).
- **کلاینت‌های منعطف**: شامل هم کلاینت پایتون (برای سرورها/NAS) و هم اسکریپت شل (ایده‌آل برای روترهای سبک مانند OpenWrt).
- **اعمال تغییرات هوشمند (Smart Diffing)**: کلاینت تنها رکوردهایی را که تغییر کرده‌اند به‌روزرسانی می‌کند، بنابراین rewriteهای شخصی AdGuard Home شما حفظ شده و از تماس‌های API غیرضروری جلوگیری می‌شود.

## 📂 ساختار پروژه

```text
.
├── client/         # اسکریپت‌های کلاینت (پایتون و شل) برای همگام‌سازی IPها با AdGuard Home
├── lists/          # لیست دامنه‌های متنی دسته‌بندی شده بر اساس سرویس
├── resolved/       # فایل‌های JSON حاوی IPهای resolve شده امن (آپدیت شده توسط GitHub Actions)
├── resolver/       # اسکریپت‌های پایتون استفاده شده توسط GitHub Actions برای انجام DoH resolution
├── README.md       # مستندات انگلیسی
└── README-fa.md    # مستندات فارسی
```

## 🚀 راهنمای شروع سریع

### پیش‌نیازها
- نصب بودن AdGuard Home روی شبکه محلی شما (مثلاً روتر، رزبری‌پای، سرور).
- **کلاینت پایتون**: نیازمند Python 3.8+
- **کلاینت شل (OpenWrt)**: نیازمند `curl` و `jq`

### ۱. نصب

مخزن را کلون کنید و وابستگی‌های لازم را نصب نمایید (در صورت استفاده از کلاینت پایتون):

```bash
git clone https://github.com/mortezabahmani/adg-dnslookup.git
cd adg-dnslookup
pip install -r requirements.txt
```

### ۲. پیکربندی

یک کپی از فایل تنظیمات نمونه ایجاد کنید:

```bash
cp client/config.example.ini client/config.ini
```

فایل `client/config.ini` را با ویرایشگر متن مورد علاقه خود ویرایش کنید:

```ini
[adguard]
url = http://192.168.1.1:3000    # آدرس URL دستگاه AdGuard Home شما
username = admin                 # نام کاربری رابط کاربری وب AdGuard Home

[github]
repo = mortezabahmani/adg-dnslookup
branch = main

[lists]
# لیست دسته‌بندی‌هایی که می‌خواهید همگام شوند، با کاما جدا شده
enabled = google,social,dev,cdn
```

**نکته امنیتی:** رمز عبور خود را به طور مستقیم در فایل config قرار ندهید. به جای آن، آن را به عنوان یک متغیر محیطی export کنید:

```bash
export AGH_PASSWORD="your_password"
```

### ۳. اجرا

توصیه می‌شود ابتدا یک اجرای تستی (dry-run) انجام دهید تا تغییرات را بدون دستکاری AdGuard Home مشاهده کنید:

```bash
python client/adg_sync.py --dry-run --verbose
```

اگر همه‌چیز درست به نظر می‌رسید، تغییرات را اعمال کنید:

```bash
python client/adg_sync.py --verbose
```

### ۴. اتوماسیون (اختیاری)

به راحتی می‌توانید یک cron job تنظیم کنید تا AdGuard Home شما به طور خودکار آپدیت بماند:

```bash
bash client/install_cron.sh
```

## 🛠 حالت‌های کاری

### حالت Online (پیش‌فرض)
آدرس‌های IP از پیش resolve شده را مستقیماً از مخزن GitHub می‌خواند. نیازمند دسترسی اینترنت به GitHub است.

```bash
python client/adg_sync.py --mode online --lists google,social
```

### حالت Standalone
اسکریپت خود به عنوان resolver عمل کرده و مخزن GitHub را نادیده می‌گیرد. دامنه‌ها را به صورت محلی با استفاده از DoH، resolve می‌کند. در صورتی که GitHub مسدود باشد اما شما به طور موقت از اتصال VPN استفاده کنید، بسیار مفید است.

```bash
python client/adg_sync.py --mode standalone --lists google,social
```

### OpenWrt / حالت سبک (Shell Script)
برای دستگاه‌هایی که امکان نصب پایتون در آن‌ها وجود ندارد.

```bash
export AGH_URL="http://192.168.1.1:3000"
export AGH_USER="admin"
export AGH_PASS="password"
export REPO="mortezabahmani/adg-dnslookup"
export LISTS="google,social,dev"

bash client/adg_sync.sh
```

## ❓ سوالات متداول (FAQ)

**آیا این روش امن است؟**
بله. IPها توسط ارائه‌دهندگان معتبر (Cloudflare، Google، Quad9) و از سرورهای امن و بدون نفوذ (GitHub Actions) resolve می‌شوند. سیستم رأی اکثریت تضمین می‌کند که اگر یک resolver نفوذ شده باشد یا خطایی برگرداند، IP نهایی همچنان دقیق باقی بماند.

**اگر GitHub مسدود باشد چه کنم؟**
از `حالت Standalone` استفاده کنید. می‌توانید لیست‌ها را از پیش دانلود کنید و فرایند resolve کردن را در حین اتصال به یک VPN مستقیماً روی دستگاه خود اجرا کنید.

**آیا این کار سرعت AdGuard Home را کاهش می‌دهد؟**
خیر. افزودن هزاران DNS Rewrite تاثیر ناچیزی بر عملکرد AdGuard Home دارد، زیرا جستجوها در آن به شدت بهینه‌سازی شده‌اند.

**آیا DNS rewriteهای دستیِ من بازنویسی خواهند شد؟**
خیر. اسکریپت تنها دامنه‌های خاصی را که در لیست‌های فعال شما موجودند مدیریت می‌کند. این مورد باعث دست‌نخورده ماندن ورودی‌های دستی AdGuard Home شما می‌شود.
